# 后续工作计划（BACKLOG）

本文档与当前版本同步，用于和 AI 或团队一起排期、拆任务。完成一项后可勾选或移走，新想法随时追加。与 AI 沟通时可直接引用下方**模块命名**，便于精准对齐。

---

## 模块命名（约定）

| 名称 | 含义 |
|------|------|
| **资产表** | 列出每个产品内容的表格（主页面上方大表） |
| **dashboard** | 汇总总资产的区域（主页面底部汇总块） |
| **功能区** | 汇集「新增产品、**− 删减产品**、**标记已清仓**、记一笔、流水列表、**已清仓产品**（`/closed-products`）、**刷新权益净值**、**刷新全部净值**、更新净值、拍瞬间、瞬间对比、产品详情」等；**底栏两行**：上行主操作，下行右对齐「放弃/保存表格修改」 |
| **瞬间** | 按当前份额与净值生成的资产定格，用于月末对比等（代码中仍用 snapshot，仅面向用户的文案用「瞬间」） |
| **产品详情** | `/products` 列表与 `/products/[id]` 详情页：基本信息、手填覆盖说明、该产品流水、最近净值（与流水列表分开展示） |

## 配置 / 约定（高优先级）

- [x] **份额与成本的关系与锁定规则**  
  - 无流水：可手填 `unitsOverride` / `costOverride`。  
  - 有流水：总览与区间盈亏仅用流水汇总份额/成本；资产表两列只读；PATCH 拒绝改覆盖。说明见 `docs/units-cost-rules.md`。

- [x] **文案统一：「快照」→「瞬间」**  
  产品内与路由/文档中面向用户的文案已改为「拍瞬间」「瞬间对比」等，代码与注释保留 `snapshot` 等英文标识。

- [x] **资产大类与展示规则（2025-03）**  
  - 一级（固定四项，无持仓也显示）：**现金**（人民币、美元、日元）、**债权**（纯债、股债混合）、**商品**、**权益**（美股、港A）。配置见 `src/lib/categories.ts`；**商品**大类分组行使用偏橙底色以便与浅底区分。  
  - **现金**一级均不维护份额；二级为**人民币**时「净值/汇率」列显示「—」；**美元、日元**时该列**优先**展示即期参考汇率（`src/lib/fx-rates.ts` → Frankfurter/ECB，`USD→CNY` / `JPY→CNY`），失败则回退产品上的记录值（DailyPrice）。  
  - 每次 `GET /api/overview` 会将历史数据 **归并**：`category` 为「美元」「日元」或 `account` 为「美元」「日元」的产品写入 **现金 + 对应二级**（事务失败仅打日志，仍返回总览）。  
  - 表头「总成本」；表头「净值/汇率」单行不换行糊格；代码列「查代码」查不到则弹窗手动输入。

## 功能（中优先级）

- [x] **手动输入数据的存储与可见性**  
  覆盖值存于 Product 的 `unitsOverride` / `costOverride`；**产品详情**页单独说明并与流水分块展示。真正「变更历史表」仍为可选后续。

- [x] **类似流水列表的新页面**  
  已增加产品详情路由与入口（总览名称链接、功能区「产品详情」、流水页互链）。

- [x] **未拍瞬间前的修改是否保存 / 撤销或暂存**  
  资产表份额与总成本改为本地草稿 + 底部「保存表格修改」批量落库；未保存离开时浏览器提示或站内弹窗（流水列表、瞬间对比、产品详情等）。

- [x] **按大类刷新净值**  
  `POST /api/refresh-prices` 支持 body `{ "category": "权益" }`（仍仅限 FUND/STOCK）。首页功能区：**刷新权益净值**（仅权益大类）、**刷新全部净值**（全库基金/股票）。理财等非标品仍走「更新净值」手填。

- [x] **资产表「净值/汇率」列：title 展示更新日期**  
  已实现：每行 `title` 由 `buildNavRateTitle` 生成；基金/股票等为最新 `DailyPrice` 的日期（`overview` 返回 `latestPriceDate`）；即期汇率为 Frankfurter 的 `date`（`fxSpotAsOfDate`）；表头列有提示「悬停单元格查看日期」。

- [x] **产品删减与清仓（两类场景）**（首版）  
  - **场景 A**：`Product.deletedAt` 软删；**「− 删减产品」**弹窗选产品 → `DELETE /api/products/[id]`，**仅无流水**时可删；`code` 置空避免 unique 冲突。已删产品详情 404。  
  - **场景 B**：`Product.closedAt`；**「标记已清仓」**→ `POST /api/products/[id]/close`；总览 / 产品下拉 / 刷新净值 / 拍瞬间 / period-pnl 等均排除 `closedAt` 非空。功能区 **「已清仓产品」** → **`/closed-products`**，列：清仓日、名称、大类/小类、末笔卖出日、卖出累计、实现盈亏（`sumLifetimeRealizedPnl`，与总览卖出成本口径一致）。产品详情页对已清仓展示提示条。  
  - **统计**：dashboard 增加 **「本月实现（卖/分红）」**（`monthRealizedPnl`，自然月内、**含已清仓流水**）；原「本月盈亏」改名为 **「本月盈亏（持仓）」**（仍相对月初瞬间，仅当前持仓市值）。瞬间对比 / 年度 review 等待后续与 `monthRealizedPnl` 对齐文档。

## 使用方式

1. **排期**：按优先级（BUG/配置 → 功能 → UX）选 1～2 项作为当前迭代重点。  
2. **拆任务**：大项拆成小步，便于分次实现与验收。  
3. **与 AI 协作**：新对话时带上「当前要做的一两件事」+ 本文档 + `PROJECT-PROGRESS.md`，并引用「资产表 / dashboard / 功能区 / 瞬间」等命名。  
4. **更新**：做完的勾选或移走，新想法写入对应优先级区块，保持 BACKLOG 为单一事实来源。
