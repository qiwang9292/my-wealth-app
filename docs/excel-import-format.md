# Excel / CSV 导入 · 格式与幂等规则

## 列名（第一行表头）

应用会按列名**模糊匹配**下列字段（中英文均可混用）：

| 含义 | 可识别列名示例 |
|------|----------------|
| 账户 | 账户、账户名称、account |
| 资产/财产类型 | 财产类型、资产类型、类型、assetType |
| 产品名称 | 产品、产品名称、name（必填） |
| 风险 | 风险级别、风险等级、风险、riskLevel |
| 数额 | 数额、金额、市值、净值、price、amount（**可留空**：须在导入弹窗按行补全后再导入）。**含义因大类而异**：权益/债权/商品多为 **最新净值（单价）**（有代码可点「查最新价」）；现金·**美元/日元**为 **外币余额**（总市值由服务端按即期汇率折算人民币）；现金·人民币、理财为 **人民币余额或当前总金额**（不是历史成本单价）。写入当天 `DailyPrice`。 |
| 代码 | 代码、基金代码、股票代码、code |
| 大类 | 大类、category（与系统一致时可手填：现金、债权、商品、权益、**理财**） |
| 细分 | 细分、子类、subCategory（随大类，如理财下：活期、定期） |
| 到期日 | 到期日、到期、maturityDate、maturity（可选；**理财·定期**时写入产品到期日） |
| 份额 | 份额、持仓份额、units（可选；须与「买入净值」**同填或同空**） |
| 买入净值 | 买入净值、单位净值、buyNav（可选；与「份额」同填时，**总成本**按 份额×买入净值 写入无流水时的覆盖字段） |

- **内部产品类型**（FUND/STOCK 等）由 **大类、细分、代码** 自动推断，导入表**无需**再填「产品类型」列。
- 建议「当日单价/市值」用列名 **数额** 或 **金额**；若用「净值」作列名，勿与 **买入净值** 列混用同一表头（两列请分开命名）。
- 仅读取 **第一个工作表**（`.xlsx`）；`.csv` 按首行表头解析。
- **模板下载**：浏览器打开或保存 [`/api/import-excel/template`](/api/import-excel/template)（带 UTF-8 BOM 的 CSV 列头；列顺序：账户、财产类型、产品、风险级别、数额、代码、大类、细分、到期日、份额、买入净值）。

## 导入行为（幂等与冲突）

1. **匹配现有产品**（用于判断更新或跳过）  
   - 若行内填了 **代码**：按 `code` 全库查找（含已删除、已清仓记录，用于判定跳过原因）。  
   - 同时会按 **名称 + 账户** 查找（`account` 空则匹配 `account` 为 null 的产品）。

2. **将更新**  
   - 匹配到 **未删除且未清仓** 的产品：更新产品字段，并对 **当天** 写入/覆盖一条 `DailyPrice`（数额列）。

3. **将新增**  
   - 无匹配活动产品时新建产品，并写入当天 `DailyPrice`。

4. **将跳过（不写入）**  
   - 与**文件中上一行**重复（同一代码，或同名同账户）：只处理第一行。  
   - 匹配到 **已清仓** 产品。  
   - 匹配到 **已软删** 产品。  
   - **代码** 已被任意产品占用（含已删除/清仓），与新建冲突。

5. **导入成功自动拍瞬间**  
   - 若有任意一行成功 **新增或更新**，服务端在写库结束后自动创建一条瞬间，备注为 `Excel 导入自动生成`（与手拍瞬间同一套估值口径，见 `src/lib/valuation-snapshot.ts`）。

## 预检与错误导出

- 先 **预检** 再 **开始导入**：预检返回「将新增 / 将更新 / 将跳过」及解析错误行列表。  
- 若某行 **未填数额** 且该行将实际导入（非「将跳过」），预检会列出 **须补数额** 行；在弹窗表格中填好后，**开始导入** 时客户端会把补全值随请求提交（`supplements`，键为 Excel **行号**，与预检「第 N 行」一致）。  
- 解析错误可点击 **导出错误 CSV**（预检结果区内）。

## 与流水的关系

- 导入 **不会** 创建流水；份额/成本仍遵循 `docs/units-cost-rules.md`。
- 若行内填写了 **份额 + 买入净值**：新建产品时会写入 `unitsOverride` / `costOverride`（总成本自动算）；**更新**已有产品时，仅当该产品 **尚无任何流水** 时才覆盖这两项，有流水则忽略列内份额/净值以免与账本冲突。
